# TON-Eats Design Context

## Project Overview
TON-Eats is a decentralized, 0% commission food delivery network powered by the TON blockchain. The platform integrates seamlessly with Telegram Mini Apps and features on-chain escrow, real-time tracking, and Web3 native payments.

## Design Context

### Users
- **Primary**: Telegram Mini App users (mobile-first, prefer minimal setup friction)
- **Secondary**: Premium restaurant partners (expect professional, polished experience)
- **Tertiary**: Delivery couriers (need clear, functional interfaces for logistics)

**Job to be Done**: 
- Customers: Order food with blockchain security & instant transparent pricing
- Merchants: Receive 100% commission payouts without intermediaries
- Couriers: Manage deliveries with real-time GPS tracking

### Brand Personality
**Trustworthy, Decentralized, Premium**

The platform combines Web3 innovation with high-end UX. Users should feel confidence in blockchain security while experiencing Apple-level polish and refinement. The brand is modern, forward-thinking, and respects user autonomy.

### Aesthetic Direction
- **Visual Tone**: Premium, minimal, modern
- **Reference**: Apple (polished, spacious, intentional)
- **Color Palette**: TON-Eats Orange accent (#FF9500) on clean neutral backgrounds
- **Light & Dark Modes**: Both fully supported with smooth transitions
- **Typography**: Inter (modern geometric sans-serif)
- **Anti-References**: Garish, cluttered, or overly playful food delivery apps

### Design Principles
1. **Premium Simplicity** — Remove friction; respect user intelligence. Every element must earn its space.
2. **Mobile-First Clarity** — Design for Telegram Mini App constraints. Touch-friendly, scannable, breathable.
3. **Trust Through Transparency** — Make blockchain benefits (fees, escrow, payouts) immediately visible and understandable.
4. **Delightful Polish** — Smooth transitions, consistent spacing, refined micro-interactions. Details matter.
5. **Accessible by Default** — WCAG AA minimum; test with real accessibility tools.

### Current Tech Stack
- **Frontend**: React (Vite), TypeScript, Styled-Components, Material UI
- **Design System**: CSS custom properties (color scale, transitions, elevations, spacing)
- **Existing Assets**: Orange accent, Inter font, dark mode support

### Known Pain Points to Address
1. Checkout/payment flow confusion
2. Mobile responsiveness inconsistencies
3. Inconsistent component styling (isolation)
4. Unclear UX copy/labels/instructions
5. Poor visual hierarchy in some flows
6. Missing accessibility features (ARIA, focus states, etc.)
7. Performance issues with animations/rendering
